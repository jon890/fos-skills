# Step 5 — API/함수 설계

## 목적

필요한 엔드포인트·함수 목록과 요청/응답 스키마 초안을 확정한다.

## 필수 산출물

- 엔드포인트·함수 목록 + 요청/응답 스키마 초안.
- 기존 엔드포인트·함수 **재사용 가능 여부**.
- 도메인 변형은 오버레이 참조.

## 캐낼 모호점 (능동 발굴 → 사용자 확정)

- 응답 스키마의 필드·옵셔널·에러 코드가 불명확한가.
- 새로 만드느냐 기존을 고치느냐 갈리는 지점.
- 경로·시그니처 네이밍(스코프 파라미터 위치 등)이 애매한가.
- 동기 vs 비동기(잡·폴링) 여부.
- 페이지네이션·필터·정렬 필요 여부.

## 게이트 통과

엔드포인트·함수·스키마가 정해지고 모호점 **0** → 6단계.
