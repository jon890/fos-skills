# Step 4 — 인터페이스 설계

## 목적

사용자·호출자가 마주하는 표면을 설계한다.

## 필수 산출물

- 도메인 변형은 오버레이 참조:
    - UI = 화면별 정보·컴포넌트 구조·상태.
    - CLI = 명령 시그니처·플래그·출력 포맷.
    - backend = 엔드포인트별 요청/응답 스키마 (5단계와 겹치면 병합).

## 캐낼 모호점 (능동 발굴 → 사용자 확정)

- **UI 레이아웃·컴포넌트 분해가 정해졌나** — 화면 구성, 어떤 컴포넌트, 어디 배치.
- 입력 방식(직접 입력 vs 선택기 vs 자동)이 갈리는 지점.
- 표면의 명명(경로·라벨·필드명)이 애매한가.
- 상태 전이(로딩·성공·실패 표시)를 어떻게 보여주나.

## 게이트 통과

인터페이스 표면이 구체화되고 모호점 **0** → 5단계.
