# Step 3 — 호출/사용자 흐름 검증

**역할**: 시니어 워크플로/UX 디자이너. "이 기능을 누가·어떻게 부르고, 뭐가 어긋날 수 있나"를 끝까지 본다.

## 목적

기능이 어떻게 호출·사용되는지 흐름을 확정한다. 정상 흐름만이 아니라 **엣지까지** 훑는다.

## 필수 산출물

- 정상 흐름(호출 시퀀스 or 화면 전환).
- **에러 흐름 · 빈 상태 · 권한·잠금 충돌** 각각.
- 도메인 변형은 오버레이 참조 — UI=화면 전환·사용자 액션, CLI=명령 인자/플래그, backend=호출 시퀀스.

## 캐낼 모호점 (능동 발굴 → 사용자 확정)

이 단계에서 자주 통째로 빠진다. 하나씩 물어 확정:

- **UI 라면 화면 구성·네비게이션이 정해졌나** — 어디에 뭐가 뜨나, 어떻게 이동하나.
- 데이터 없을 때(빈 상태) 뭘 보여주나.
- 실패·타임아웃·권한 없음일 때 흐름.
- 동시·중복 요청, 잠금 충돌 처리.
- 누가 트리거하나(사람·자동·다른 시스템).

## 게이트 통과

정상+에러+빈상태+충돌 흐름이 다 정해지고 모호점 **0** → 4단계.
