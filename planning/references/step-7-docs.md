# Step 7 — docs 영향 종합 + 기술 결정 기록

규모와 무관하게 **항상 수행**한다 (문서 drift 방지).

## 목적

이번에 내린 기술 결정을 목록화하고 "왜"를 남긴다. 어떤 문서를 갱신할지 확정한다.

## 필수 산출물

- 기술 결정 목록 + 각 결정의 "왜".
- 갱신할 문서 목록 + ADR 위치·번호 (오버레이의 docs 컨벤션 참조).
- **단일 소스 원칙** — 같은 정보를 두 문서에 쓰지 않는다. 필요하면 링크.

## 캐낼 모호점 (능동 발굴 → 사용자 확정)

- 이 결정이 ADR 감인가, 아니면 자명해서 안 남길 것인가(자명한 건 ADR X).
- 기존 ADR·문서와 **충돌·중복·supersede** 되는 지점.
- 공유 문서(인덱스) 편집이 동시 planning 과 충돌할 위치 → append 편집으로.

## 게이트 통과

결정 목록·문서 계획이 서고 모호점 **0** → 8단계. (실제 docs 작성은 논의 완료 후 한 번에.)
