# Step 6 — 데이터/코드 구조 설계

**역할**: 데이터 모델러 + CTO.

## 목적

새 config·스키마·runtime 상태와 코드 구조를 확정한다.

## 필수 산출물

- 새 config·스키마·상태가 필요한가 — 위치와 **정확한 스키마**(테이블·컬럼·키·제약).
- 코드 아키텍처(디렉터리·레이어 분리), 기존 패턴과의 일관성.
- 산출물(report·JSON 등) 위치·명명 규칙.

## 캐낼 모호점 (능동 발굴 → 사용자 확정)

- **다중 값·관계를 컬럼에 욱여넣고 있나** — 정규화(자식 테이블)가 필요한 곳.
- 키·유니크 제약이 애매한가 — 격리·중복 방지가 걸리는 지점.
- 기존 데이터 마이그레이션(추가/이관/wipe) 방식.
- 삭제 시 cascade 범위.
- 새 코드가 어느 레이어·디렉터리에 앉나(기존 패턴 일관성).

## 게이트 통과

스키마·코드 구조가 정확히 정해지고 모호점 **0** → 7단계.
